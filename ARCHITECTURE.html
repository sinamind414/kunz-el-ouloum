🌐 Flux global de la requête

Navigateur / App Android ➜ server.ts (Express) ➜ Vite middleware (dev) / dist/ (prod) ➜ React App ➜ Modules contenu (TS embarqué) ➜ Tuteur SVT (recherche + réponse)
Aucune base de données externe, aucun appel API IA : tout le contenu est intégré (bundlé) dans le code TypeScript. Le mode « tutor » est offline.

Backend server.ts — serveur unique

Express + Viteserver.ts Démarre sur PORT (défaut 3000), écoute 0.0.0.0
En-têtes sécurité X-Frame-Options, nosniff, XSS-Protection, Permissions-Policy (faits main, pas helmet)
Route santéGET /api/health Renvoie {status, mode, tutor:"offline", dataVersion}
Mode dev middlewareMode Vite (SPA)
Mode prod Sert dist/ (cache 1 an, index.html no-cache) + catch-all *
⚠️ Pas d'API métier réelle : /api/health seulement. Le catch-all app.get("*") bloque toute extension future par API. Pas de gestionnaire d'erreurs.
